What It Is

ai-shell is a single bash function that turns every terminal on your Linux machine into a natural language interface. You describe what you want in plain English. It returns the exact command, explains it, and runs it. No manual pages. No Stack Overflow. No context switching.

It is not a chatbot. It is not a wrapper around a CLI tool. It is a direct bridge between human intent and kernel execution, powered by Claude Haiku 4.5 and backed by a persistent memory system that learns from every interaction.

~/.ai-shell.sh — 400 lines of bash. curl + jq. That's it.

Why It Matters

The Linux command line is the most powerful interface ever built. It is also the most hostile to newcomers, the most hostile to the forgetful, and the most hostile to anyone who doesn't spend their life memorizing flag combinations for 3,000 different utilities.

ai-shell eliminates the gap between knowing what you want and knowing how to say it. It doesn't dumb down the terminal — it makes the terminal's full power accessible to anyone who can form a sentence.

A sysadmin who has been using Linux for 20 years still googles tar flags. ai-shell knows them. It also knows your specific machine, your GPU driver, your filesystem layout, your package manager, and what you asked it three days ago.

How It Works

On first run, ai-shell snapshots your system: kernel version, CPU, GPU, RAM, package manager, init system, boot parameters, user groups. This context is cached daily and injected into every API call. Claude Haiku doesn't give you generic Linux answers — it gives you answers for your machine.

Every interaction is logged as a JSON chunk: timestamp, working directory, query, command, explanation, and truncated output. Before each new query, the memory system bundles the last 5 interactions (short-term recall) and keyword-searches the full history for relevant past interactions (long-term recall). The model sees what you've done before and builds on it.

If a command fails — wrong flags, missing tool, permission denied — ai-shell detects the failure pattern, sends the error back to the model, and auto-corrects. One retry. No human intervention. The lsblk -h problem never happens twice.

After every run, the tool reads its own source code, reviews recent usage patterns, and suggests one concrete improvement to itself. It is a tool that evolves by observing how you use it.
